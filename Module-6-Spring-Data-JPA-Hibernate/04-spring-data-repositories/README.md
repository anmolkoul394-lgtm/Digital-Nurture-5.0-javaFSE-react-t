# Spring Data Repositories

## The repository hierarchy

- `Repository<T, ID>` - a marker interface, no methods
- `CrudRepository<T, ID>` - adds `save()`, `findById()`, `findAll()`, `deleteById()`, `count()`
- `PagingAndSortingRepository<T, ID>` - adds paged/sorted `findAll()`
- `JpaRepository<T, ID>` - adds JPA-specific extras (batch operations, flushing) - this is
  the one you'll extend in almost every real project

```java
public interface BookRepository extends JpaRepository<Book, Long> {
}
```

That's a complete, working repository - Spring generates the implementation for you at
startup, using a dynamic proxy behind the scenes.

## Derived query methods

You can also declare extra finder methods just by following a naming convention, and
Spring parses the method name into an actual query - covered properly in
`06-query-methods-named-queries`, but a quick preview:

```java
List<Book> findByTitleContaining(String keyword);
```

## Custom query methods with @Query

For anything the naming convention can't express cleanly, `@Query` lets you write JPQL
directly on the method:

```java
@Query("SELECT b FROM Book b WHERE b.price > :minPrice")
List<Book> findExpensiveBooks(@Param("minPrice") double minPrice);
```
