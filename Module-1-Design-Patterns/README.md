## 🎯 Why Design Patterns?

Design patterns are proven, reusable solutions to common software design problems. They make code:

- Easier to **maintain**
- Easier to **extend** without breaking existing behaviour
- Easier for other developers to **understand** (shared vocabulary)

## 📖 Categories of GoF Patterns

- **Creational** – deal with object creation (Singleton, Factory Method, Builder)
- **Structural** – deal with composing classes/objects (Adapter, Decorator, Proxy)
- **Behavioral** – deal with communication between objects (Observer, Strategy, Command)
